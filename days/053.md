# Resolve VolumeMounts Issue in Kubernetes

We encountered an issue with our Nginx and PHP-FPM setup on the Kubernetes cluster this morning, which halted its functionality. Investigate and rectify the issue:

- The pod name is `nginx-phpfpm` and configmap name is `nginx-config`. Identify and fix the problem.

- Once resolved, copy `/home/thor/index.php` file from the jump host to the `nginx-container` within the nginx document root. After this, you should be able to access the website using Website button on the top bar.

> Note: The kubectl utility on jump_host is configured to operate with the Kubernetes cluster.

## Steps

1. Let's get the pod yaml config:

    ```sh
    kubectl get pod nginx-phpfpm -o yaml > nginx-phpfpm.yml
    ```

2. Let's fix the yaml file:

    ```sh
    vi nginx-phpfpm.yml
    ```

    **The Issue:** There are two containers (nginx and php-fpm) that share a volume called `shared-files`. However, they are mounted at different paths:
    - php-fpm container mounts at: `/var/www/html`
    - nginx container mounts at: `/usr/share/nginx/html`

    To confirm the correct mount path, check the nginx-config configmap:

    ```sh
    kubectl get configmap nginx-config -o yaml
    ```

    **Output:**
    ```yaml
    apiVersion: v1
    data:
      nginx.conf: |
        events {
        }
        http {
          server {
            listen 8099 default_server;
            listen [::]:8099 default_server;

            # Set nginx to serve files from the shared volume!
            root /var/www/html;
            index  index.html index.htm index.php;
            server_name _;
            location / {
              try_files $uri $uri/ =404;
            }
            location ~ \.php$ {
              include fastcgi_params;
              fastcgi_param REQUEST_METHOD $request_method;
              fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
              fastcgi_pass 127.0.0.1:9000;
            }
          }
        }
    kind: ConfigMap
    ```

    The nginx root directory in the configmap is `/var/www/html`. So we need to update the nginx container's mountPath to match.

    In the nginx-phpfpm.yml file, find the nginx-container volumeMounts section:

    ```yaml
    - name: nginx-container
      image: nginx:latest
      volumeMounts:
      - mountPath: /usr/share/nginx/html    # ❌ WRONG - needs to be changed
        name: shared-files
      - mountPath: /etc/nginx/nginx.conf
        name: nginx-config-volume
        subPath: nginx.conf
    ```

    Change it to:

    ```yaml
    - name: nginx-container
      image: nginx:latest
      volumeMounts:
      - mountPath: /var/www/html             # ✅ CORRECT - matches nginx config
        name: shared-files
      - mountPath: /etc/nginx/nginx.conf
        name: nginx-config-volume
        subPath: nginx.conf
    ```

    Also, remove runtime fields from the YAML file (status, creationTimestamp, resourceVersion, uid, managedFields, nodeName, hostname, etc.) to avoid errors when recreating.

3. Let's delete the existing pod and recreate with the fixed configuration:

    ```sh
    kubectl delete pod nginx-phpfpm
    kubectl apply -f nginx-phpfpm.yml
    ```

    Wait for the pod to be ready:

    ```sh
    kubectl get pods
    ```

4. Now, let's copy `index.php` to the nginx document root:

    ```sh
    kubectl cp /home/thor/index.php nginx-phpfpm:/var/www/html/index.php -c nginx-container
    ```

5. Verify the file was copied successfully:

    ```sh
    kubectl exec nginx-phpfpm -c nginx-container -- ls -la /var/www/html/
    ```

6. Check if the service is routing traffic properly:

    ```sh
    kubectl get svc nginx-service
    kubectl get endpoints nginx-service
    ```

    If the endpoints show `<none>`, check the service selector:

    ```sh
    kubectl describe svc nginx-service
    ```

    The service selector should match the pod labels. Update if needed:

    ```sh
    kubectl edit svc nginx-service
    ```

    Change the selector to match the pod labels (app: nginx-phpfpm or app: php-app).

7. Test the website:

    ```sh
    # Test internally
    kubectl exec nginx-phpfpm -c nginx-container -- curl http://localhost:8099/index.php

    # Test through the service
    kubectl run test-pod --image=curlimages/curl --rm -it --restart=Never -- curl http://nginx-service:8099/index.php
    ```

8. Access the website using the **Website** button on the top bar. The website should now be accessible!

## Good to Know?

### Volume Troubleshooting

- **Mount Path Mismatch**: Containers must share same mount path
- **ConfigMap Issues**: Verify ConfigMap exists and is correctly named
- **Permission Problems**: Check file system permissions
- **Volume Types**: emptyDir, hostPath, PVC, ConfigMap, Secret

### Multi-Container Pods

- **Shared Volumes**: Containers share volumes within pod
- **Sidecar Pattern**: Helper containers alongside main application
- **Data Sharing**: Share files between containers
- **Log Aggregation**: Collect logs from multiple containers

### ConfigMap Volumes

- **Configuration Files**: Mount config files into containers
- **subPath**: Mount specific files from ConfigMap
- **File Mode**: Set file permissions for mounted files
- **Updates**: ConfigMap changes reflect in mounted volumes

### Debugging Techniques

- **Describe Pod**: `kubectl describe pod` shows events and issues
- **Pod Logs**: `kubectl logs pod-name -c container-name`
- **Exec into Pod**: `kubectl exec -it pod-name -c container -- /bin/sh`
- **Volume Inspection**: Check mount points and file contents
- **Service Endpoints**: `kubectl get endpoints` to verify service connectivity
