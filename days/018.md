# Day 18: Install and Configure MariaDB Server

## Objective

Install MariaDB, create databases, users, and assign privileges.

## Prerequisites

- SSH access to `stdb01` as user `peter`
- Sudo privileges
- Internet access for package installation

## Steps Performed

### 1. Connect to the Database Server
```bash
ssh peter@stdb01
```

### 2. Switch to Root User
```bash
sudo -i
```

### 3. Install MariaDB Server
```bash
yum install mariadb-server -y
```

### 4. Enable and Start MariaDB Service
```bash
systemctl enable mariadb
systemctl start mariadb
```

### 5. Login to MariaDB
```bash
mysql
```

### 6. Create Database
```sql
CREATE DATABASE kodekloud_db8;
```

### 7. Create Database User
```sql
CREATE USER 'kodekloud_joy'@'localhost' IDENTIFIED BY 'LQfKeWWxWD';
```

### 8. Grant All Privileges
```sql
GRANT ALL PRIVILEGES ON kodekloud_db8.* TO 'kodekloud_joy'@'localhost';
FLUSH PRIVILEGES;
```

### 9. Verify Configuration
```sql
SHOW DATABASES;
SELECT User, Host FROM mysql.user;
SHOW GRANTS FOR 'kodekloud_joy'@'localhost';
```

## Good to Know

### 🔐 Security & Authentication
- **Default root password is empty** - Always run `mysql_secure_installation` after installation
- **Password hashing** - MariaDB uses `PASSWORD()` function; passwords are stored as hashes in `mysql.user` table
- **Authentication plugin** - Modern MariaDB uses `unix_socket` or `mysql_native_password` authentication
- **Reset root password** - Use `--skip-grant-tables` option if root password is lost

### 📊 Database Management
- **Case sensitivity** - Database and table names are case-sensitive on Linux (not on Windows)
- **Character sets** - Default is `latin1`; recommend `utf8mb4` for full Unicode support
- **Storage engines** - InnoDB (transactional) vs MyISAM (faster reads, no transactions)
- **Database size limit** - Depends on filesystem; InnoDB can handle up to 64TB
