# Day 001: Create User with Non-Interactive Shell

**Difficulty:** 🟢 Beginner
**Time:** 10 Minutes
---

## 🎯 Objective

The system admin team at xFusionCorp Industries requires the creation of a user account with a non-interactive shell. This type of account is commonly used for service accounts, backup agents, and automated processes where direct login access is not required.

Your task is to create a user named `ammar` with a non-interactive shell on **App Server 3**.

---

## 📋 Prerequisites

* Access to a Linux server (CentOS / Ubuntu / RHEL)
* SSH access to the target server
* `sudo` privileges
* Basic knowledge of Linux user management

---

## 🛠️ Technologies Used

* Linux User Management
* SSH
* System Administration Commands

---

# 🧩 Task

Create a user named `ammar` with a non-interactive shell on **App Server 3**.

You can use the server credentials available from the infrastructure details page:

[KodeKloud Engineer – Infrastructure Details](https://kodekloudhub.github.io/kodekloud-engineer/docs/projects/nautilus?utm_source=chatgpt.com#infrastructure-details)

---

# ✅ Solution

## Step 1: Login to App Server 3

```bash
ssh banner@stapp03
```

> Use the password provided in the infrastructure details/password list.

---

## Step 2: Create the User with Non-Interactive Shell

```bash
sudo useradd -s /sbin/nologin ammar
```

If `/sbin/nologin` is unavailable on your system, use:

```bash
sudo useradd -s /usr/sbin/nologin ammar
```

---

## Step 3: Verify the User

```bash
grep ammar /etc/passwd
```

### Expected Output

```bash
ammar:x:1002:1002::/home/ammar:/sbin/nologin
```

---

# 🧠 Explanation

| Command Part    | Description                       |
| --------------- | --------------------------------- |
| `useradd`       | Creates a new Linux user          |
| `-s`            | Specifies the login shell         |
| `/sbin/nologin` | Prevents interactive login access |
| `ammar`         | Username being created            |

---

# 🚀 Final Command

```bash
sudo useradd -s /sbin/nologin ammar
```

---

# 🔍 Verification & Troubleshooting

## Common Issues

### Permission Denied

Ensure your user has `sudo` privileges.

### User Already Exists

Check existing users:

```bash
cat /etc/passwd | grep ammar
```

### Shell Not Found

Verify the `nologin` binary exists:

```bash
which nologin
```

---

# 📌 Useful Commands

## List All Users with Non-Interactive Shell

```bash
grep nologin /etc/passwd
```

## Check User Details

```bash
id ammar
```

## Remove User (If Needed)

```bash
sudo userdel -r ammar
```

---

# 🔐 Security Best Practices

* Service accounts should use non-interactive shells.
* Regular users should use interactive shells like `/bin/bash`.
* Follow the principle of least privilege.
* Always verify user creation after execution.

---

# 📚 Good to Know

## Linux User Types

* Regular Users
* System Users
* Service Accounts

## Common Shell Types

| Shell               | Purpose                 |
| ------------------- | ----------------------- |
| `/bin/bash`         | Interactive login shell |
| `/usr/sbin/nologin` | Non-interactive shell   |
| `/bin/false`        | Deny all shell access   |

## Important System Files

| File          | Purpose                         |
| ------------- | ------------------------------- |
| `/etc/passwd` | Stores user account information |
| `/etc/shadow` | Stores encrypted passwords      |

---

# 🛠️ Useful `useradd` Options

| Option | Description           |
| ------ | --------------------- |
| `-m`   | Create home directory |
| `-s`   | Specify login shell   |
| `-d`   | Custom home directory |
| `-g`   | Primary group         |
| `-G`   | Additional groups     |
| `-e`   | Account expiry date   |

---

# 🎯 Key Takeaways

* Non-interactive shells improve system security.
* Service accounts should not have login access.
* `nologin` is preferred for automation/service users.
* Understanding Linux user management is essential for system administrators.
