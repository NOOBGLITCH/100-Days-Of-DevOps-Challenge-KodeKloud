# Fix Python Application Deployment on Kubernetes

The Nautilus DevOps team found that an existing Python Flask application was not accessible after deployment. After troubleshooting, we identified and fixed the deployment and service configuration so the application became accessible through the required NodePort.

## Requirements

* Fix the existing deployment **`python-deployment-nautilus`**
* Use image **`poroko/flask-demo-app`**
* Ensure the application is running successfully
* Configure the existing NodePort Service
* NodePort: **32345**
* TargetPort: **5000** (Flask default port)

---

# Steps We Performed

## 1. Verify the Deployment

```bash
kubectl get deployments
kubectl describe deployment python-deployment-nautilus
```

We found the deployment was using an incorrect image:

```text
Image: poroko/flask-app-demo
```

The correct image should be:

```text
poroko/flask-demo-app
```

---

## 2. Fix the Deployment Image

```bash
kubectl set image deployment/python-deployment-nautilus \
python-container-nautilus=poroko/flask-demo-app
```

---

## 3. Wait for the Rollout

```bash
kubectl rollout status deployment/python-deployment-nautilus
```

Expected output:

```text
deployment "python-deployment-nautilus" successfully rolled out
```

---

## 4. Verify the Service

```bash
kubectl describe svc python-service-nautilus
```

We found the service was exposing the wrong port:

```text
Port:        8080/TCP
TargetPort:  8080/TCP
NodePort:    32345/TCP
```

Since Flask listens on **5000**, the service needed to be updated.

---

## 5. Update the Service

```bash
kubectl edit svc python-service-nautilus
```

Replace the ports section with:

```yaml
ports:
- protocol: TCP
  port: 5000
  targetPort: 5000
  nodePort: 32345
```

Alternatively, patch the service:

```bash
kubectl patch svc python-service-nautilus --type='json' -p='[
{"op":"replace","path":"/spec/ports/0/port","value":5000},
{"op":"replace","path":"/spec/ports/0/targetPort","value":5000}
]'
```

---

## 6. Verify the Deployment

```bash
kubectl get deployments

kubectl get pods
```

Expected:

```text
NAME                           READY
python-deployment-nautilus     1/1

NAME                                              READY   STATUS
python-deployment-nautilus-xxxxxxxxxx-xxxxx       1/1     Running
```

---

## 7. Verify the Service

```bash
kubectl get svc

kubectl describe svc python-service-nautilus
```

Expected:

```text
NAME                      TYPE       PORT(S)
python-service-nautilus   NodePort   5000:32345/TCP
```

---

## 8. Verify the Endpoints

```bash
kubectl get endpoints python-service-nautilus
```

Expected:

```text
NAME                      ENDPOINTS
python-service-nautilus   <pod-ip>:5000
```

---

# Good to Know?

### Kubernetes Deployment

* Manages Pods and ensures the desired number of replicas are running.
* Supports rolling updates with zero or minimal downtime.

### Rolling Update

* Updates Pods incrementally without deleting the entire application.
* Automatically replaces old Pods with new ones.

### NodePort Service

* Exposes an application on a fixed port across all Kubernetes nodes.
* Makes applications accessible from outside the cluster.

### Flask Default Port

* Flask applications listen on **port 5000** by default.
* Kubernetes Services must forward traffic to the correct application port using `targetPort`.
