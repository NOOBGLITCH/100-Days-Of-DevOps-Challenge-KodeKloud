# Create Kubernetes Secret and Mount it in a Pod

The Nautilus DevOps team needed to create a Kubernetes Secret from an existing file and mount it inside a Pod.

## Requirements

* Create a generic Secret named **`official`** from `/opt/official.txt`
* Create a Pod named **`secret-devops`**
* Container name: **`secret-container-devops`**
* Image: **`debian:latest`**
* Keep the container running using the `sleep` command
* Mount the Secret at **`/opt/apps`**

---

# Steps We Performed

## 1. Create the Secret

```bash
kubectl create secret generic official \
  --from-file=/opt/official.txt
```

Verify:

```bash
kubectl get secrets
```

Expected:

```text
NAME        TYPE
official    Opaque
```

---

## 2. Create the Pod

Create **`secret-pod.yaml`**

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secret-devops
spec:
  containers:
  - name: secret-container-devops
    image: debian:latest
    command:
    - sleep
    - "3600"
    volumeMounts:
    - name: official-secret
      mountPath: /opt/apps
      readOnly: true
  volumes:
  - name: official-secret
    secret:
      secretName: official
```

Apply it:

```bash
kubectl apply -f secret-pod.yaml
```

---

## 3. Verify the Pod

```bash
kubectl get pods
```

Expected:

```text
NAME             READY   STATUS
secret-devops    1/1     Running
```

---

## 4. Verify the Secret Inside the Container

```bash
kubectl exec -it secret-devops -c secret-container-devops -- ls /opt/apps
```

Expected:

```text
official.txt
```

Display the secret:

```bash
kubectl exec -it secret-devops -c secret-container-devops -- cat /opt/apps/official.txt
```

It should display the password/license number stored in the file.

# Quick Commands

```bash
kubectl create secret generic official --from-file=/opt/official.txt
```

## Good to Know?

### Kubernetes Secrets

- **Purpose**: Store sensitive information (passwords, tokens, keys)
- **Base64 Encoding**: Data is base64 encoded (not encrypted)
- **Namespace Scoped**: Secrets belong to specific namespace
- **Volume Mounts**: Mount secrets as files in containers

### Secret Types

- **generic**: Arbitrary user-defined data
- **docker-registry**: Docker registry credentials
- **tls**: TLS certificates and keys
- **service-account-token**: Service account tokens

### Secret Usage

- **Environment Variables**: Inject as env vars
- **Volume Mounts**: Mount as files in filesystem
- **Image Pull Secrets**: Authenticate to private registries
- **Service Account**: Automatic token injection

### Security Best Practices

- **RBAC**: Control access to secrets
- **Encryption at Rest**: Enable etcd encryption
- **Rotation**: Regularly rotate secrets
- **Least Privilege**: Grant minimal required access
- **External Systems**: Consider external secret management
