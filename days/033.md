# Resolving Git Rebase Conflicts and Pushing Changes

## Objective

A developer attempted to push changes to a remote repository but encountered a rejection because the remote repository contained newer commits. The goal is to update the local repository, resolve any conflicts, and successfully push the changes.

---

# Steps

## 1. Login to the Storage Server

```bash
ssh max@ststor01
```

Password:

```text
Max_pass123
```

Move to the repository:

```bash
cd ~/story-blog
```

---

## 2. Attempt to Push Changes

```bash
git push origin master
```

Output:

```text
! [rejected] master -> master (fetch first)
```

This indicates that the remote repository contains commits that are not available locally.

---

## 3. Download Remote Changes

Fetch and rebase the latest changes from the remote repository:

```bash
git pull --rebase origin master
```

Git will attempt to replay local commits on top of the latest remote commits.

---

## 4. Resolve Merge Conflicts

If a conflict occurs, Git will stop the rebase process.

Check repository status:

```bash
git status
```

Open the conflicted file:

```bash
cat story-index.txt
```

Example conflict:

```text
<<<<<<< HEAD
1. The Lion and the Mouse
2. The Frogs and the Ox
=======
1. The Lion and the Mooose
2. The Frogs and the Ox
>>>>>>> feature
```

---

## 5. Fix the Conflict

Edit the file:

```bash
vi story-index.txt
```

Final correct content:

```text
1. The Lion and the Mouse
2. The Frogs and the Ox
3. The Fox and the Grapes
4. The Donkey and the Dog
```

Important:

* Remove all conflict markers.
* Ensure all four story titles exist.
* Correct the typo:

```text
Mooose → Mouse
```

---

## 6. Stage the Resolved File

```bash
git add story-index.txt
```

Verify:

```bash
git status
```

---

## 7. Continue the Rebase

```bash
git rebase --continue
```

If Vim opens:

```text
:wq
```

Git will finish replaying the remaining commits.

---

## 8. Verify Repository State

```bash
git status
```

Expected:

```text
nothing to commit, working tree clean
```

Review commit history:

```bash
git log --oneline
```

---

## 9. Push Changes

```bash
git push origin master
```

Authenticate if prompted:

```text
Username: max
Password: Max_pass123
```

Push should complete successfully.

# Good to Know

### Good to Know?

#### Push Rejected

* Remote branch has newer commits than local branch.
* Git blocks push to prevent overwriting others' work.
* Pull or rebase before pushing again.

#### Git Rebase

* Replays your commits on top of the latest branch.
* Keeps commit history linear and clean.
* Avoids unnecessary merge commits.

#### Merge Conflicts

* Occur when the same file is modified in multiple places.
* Git cannot decide which version to keep.
* Requires manual resolution by the developer.

#### Conflict Markers

* `<<<<<<< HEAD` → Current local changes.
* `=======` → Separator between versions.
* `>>>>>>> branch-name` → Incoming changes.
* Remove markers after resolving conflicts.

#### Conflict Resolution

* Open the conflicted file.
* Keep the correct content.
* Remove conflict markers.
* Stage the file with `git add`.
* Continue using `git rebase --continue`.

#### Rebase Commands

* `git pull --rebase origin master` → Update local branch.
* `git rebase --continue` → Continue after fixing conflicts.
* `git rebase --abort` → Cancel the rebase.
* `git status` → Check rebase progress.

#### Verification

* Ensure all required changes exist.
* Check commit history using `git log --oneline`.
* Confirm working tree is clean.
* Push changes successfully to remote.

#### Best Practices

* Pull frequently to stay updated.
* Make small and focused commits.
* Communicate with teammates on shared files.
* Use feature branches for development.
* Review conflicts carefully before resolving.
