# Perform a Rolling Update in Kubernetes

The Nautilus DevOps team needed to perform a rolling update on an existing Deployment by upgrading the Nginx image to **`nginx:1.18`** while ensuring zero downtime.

## Requirements

* Deployment name: **`nginx-deployment`**
* Update image to **`nginx:1.18`**
* Ensure all Pods are running after the update

---

# Steps We Performed

## 1. Verify the Deployment

```bash
kubectl get deployments
```

View Deployment details:

```bash
kubectl describe deployment nginx-deployment
```

---

## 2. Check the Container Name

Before updating the image, identify the container name used in the Deployment.

```bash
kubectl get deployment nginx-deployment -o jsonpath='{.spec.template.spec.containers[*].name}'; echo
```

Example output:

```text
nginx-container
```

> **Note:** The container name is not always `nginx`. Use the name returned by the command in the next step.

---

## 3. Perform the Rolling Update

Update the image using the correct container name.

Example:

```bash
kubectl set image deployment/nginx-deployment nginx-container=nginx:1.18
```

If your container is named `nginx`, then use:

```bash
kubectl set image deployment/nginx-deployment nginx=nginx:1.18
```

Expected output:

```text
deployment.apps/nginx-deployment image updated
```

---

## 4. Monitor the Rollout

```bash
kubectl rollout status deployment/nginx-deployment
```

Expected:

```text
deployment "nginx-deployment" successfully rolled out
```

---

## 5. Verify the Pods

```bash
kubectl get pods
```

All Pods should be in the **Running** state.

---

## 6. Verify the Updated Image

```bash
kubectl get deployment nginx-deployment -o=jsonpath='{.spec.template.spec.containers[0].image}'; echo
```

Expected:

```text
nginx:1.18
```

You can also verify using:

```bash
kubectl describe deployment nginx-deployment
```

---

# Quick Commands

```bash
kubectl get deployment nginx-deployment -o jsonpath='{.spec.template.spec.containers[*].name}'; echo

kubectl set image deployment/nginx-deployment <container-name>=nginx:1.18

kubectl rollout status deployment/nginx-deployment

kubectl get pods

kubectl get deployment nginx-deployment -o=jsonpath='{.spec.template.spec.containers[0].image}'; echo
```

Replace `<container-name>` with the actual container name returned in the first command.

---

# Good to Know?

### Rolling Update

* Updates Pods one at a time.
* Maintains application availability during deployment.
* Gradually replaces old Pods with new ones.

### `kubectl set image`

* Updates the container image in an existing Deployment.
* Automatically triggers a rolling update.
* Requires the correct **container name**, not just the Deployment name.

### Rollout Status

* Monitors the progress of the update.
* Confirms when all new Pods are ready.
