# Deploy Pods in Kubernetes Cluster

The Nautilus DevOps team is beginning their Kubernetes journey by deploying applications as Pods. In this task, we created a Pod running the latest Apache HTTP Server (`httpd`) image, applied the required label, and assigned a custom container name.

## Requirements

* Create a Pod named **`pod-httpd`**
* Use image **`httpd:latest`**
* Set label **`app=httpd_app`**
* Name the container **`httpd-container`**
* Verify the Pod is running

---

# Steps We Performed

## 1. Verify Cluster Connectivity

Before creating the Pod, we confirmed that the Kubernetes cluster was accessible.

```bash
kubectl get nodes
```

Expected output:

```text
NAME           STATUS   ROLES           AGE
controlplane   Ready    control-plane   ...
```

---

## 2. Create the Pod Manifest

We created a YAML file named `pod-httpd.yaml`.

```bash
vi pod-httpd.yaml
```

Paste the following configuration:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-httpd
  labels:
    app: httpd_app
spec:
  containers:
    - name: httpd-container
      image: httpd:latest
```

Save and exit.

---

## 3. Deploy the Pod

Apply the manifest:

```bash
kubectl apply -f pod-httpd.yaml
```

Expected output:

```text
pod/pod-httpd created
```

---

## 4. Verify the Pod

Check the Pod status:

```bash
kubectl get pods
```

Expected output:

```text
NAME        READY   STATUS    RESTARTS   AGE
pod-httpd   1/1     Running   0          10s
```

---

## 5. Verify the Label

```bash
kubectl get pod pod-httpd --show-labels
```

Expected output:

```text
NAME        READY   STATUS    LABELS
pod-httpd   1/1     Running   app=httpd_app
```

Or use:

```bash
kubectl get pod pod-httpd -o jsonpath='{.metadata.labels.app}'; echo
```

Expected:

```text
httpd_app
```

---

## 6. Verify the Container Name

```bash
kubectl get pod pod-httpd -o jsonpath='{.spec.containers[0].name}'; echo
```

Expected:

```text
httpd-container
```

---

# Good to Know?

### Pod

* Smallest deployable unit in Kubernetes.
* Can run one or more containers.
* Shares networking and storage between containers.

### Pod Manifest

* YAML file describing the desired state of a Pod.
* Preferred approach for declarative deployments.

### Labels

* Key-value pairs attached to Kubernetes objects.
* Used for filtering, grouping, and service selection.

### Container Name

* Unique identifier for a container within a Pod.
* Useful for debugging, logging, and multi-container Pods.

### Image Tag

* `httpd:latest` explicitly uses the latest available Apache image.
* Specifying the tag ensures predictable deployments.

### kubectl apply

* Creates a resource if it does not exist.
* Updates supported fields if the resource already exists.
* Recommended for declarative Kubernetes management.
