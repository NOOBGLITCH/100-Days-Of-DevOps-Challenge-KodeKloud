

# 📅 Day 14: Apache Service Troubleshooting on Stratos DC

## 📌 Problem Statement

The production support team of **xFusionCorp Industries** deployed monitoring tools to track services running on systems in the Stratos DC. One of the monitoring systems reported that the **Apache service was unavailable** on one of the application servers.

### Objectives:
1. Identify the faulty application server
2. Fix the Apache service so it's **up and running**
3. Ensure Apache runs on **port 5000** on **all app servers**
4. No need to deploy any code — just ensure the service is running

---

| stapp03 | `banner` | `BigGr33n` | Application Server 3 |

- **Jump Host**: `thor@jump-host`
- **OS**: CentOS Stream 9
- **Web Server**: Apache (`httpd`)

## 🔍 Steps to Identify the Faulty Server

### 1. SSH into each app server from the jump host

```bash
ssh tony@stapp01
ssh steve@stapp02
ssh banner@stapp03
```

### 2. Check Apache service status

```bash
sudo systemctl status httpd
```

### 3. Check which port Apache is configured to listen on

```bash
grep -n "^Listen" /etc/httpd/conf/httpd.conf
```

**Expected output**: `Listen 5000`

### 4. Check for port conflicts

```bash
sudo ss -tulnp | grep :5000
```

---

## 🐛 Root Cause Identified

On **stapp01**:
- Apache was configured to listen on port `5000` ✅
- But **sendmail** process was already using port `5000` ❌
- Error message: `(98)Address already in use`

```
tcp   LISTEN 0      10         127.0.0.1:5000      0.0.0.0:*    users:(("sendmail",pid=11988,fd=4))
```

---

## 🛠️ Solution Steps (Performed on stapp01)

### Step 1: Kill the process occupying port 5000

```bash
sudo kill -9 11988
```

### Step 2: Start Apache service

```bash
sudo systemctl start httpd
```

### Step 3: Enable Apache to start on boot

```bash
sudo systemctl enable httpd
```

### Step 4: Verify Apache is running

```bash
sudo systemctl status httpd
```

**Expected output**:
```
Active: active (running)
Status: "Started, listening on: port 5000"
```

### Step 5: Verify port binding

```bash
sudo ss -tulnp | grep :5000
```

**Expected output**:
```
tcp   LISTEN  0   511   0.0.0.0:5000   0.0.0.0:*   users:(("httpd",...))
```

## Good to Know?

### Process Troubleshooting

- **Service Status**: `systemctl status service-name`
- **Service Logs**: `journalctl -u service-name -f`
- **Process List**: `ps aux | grep process-name`
- **Resource Usage**: `top`, `htop`, `ps aux --sort=-%cpu`

### Port Conflict Diagnosis

- **Active Ports**: `netstat -tlnup` or `ss -tlnup`
- **Process by Port**: `lsof -i :port-number`
- **Kill Process**: `kill -9 PID` (force kill)
- **Service Stop**: `systemctl stop service-name`

### Apache/HTTP Troubleshooting

- **Configuration Test**: `httpd -t` or `nginx -t`
- **Error Logs**: `/var/log/httpd/error_log`
- **Access Logs**: `/var/log/httpd/access_log`
- **Module Status**: `httpd -M` (loaded modules)

### Common Service Issues

- **Port Conflicts**: Multiple services trying to use same port
- **Permission Issues**: Service can't access required files
- **Configuration Errors**: Syntax errors in config files
- **Resource Limits**: Insufficient memory or file descriptors
