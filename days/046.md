# Deploy an App on Docker Containers

The Nautilus Application Development team requested a two-tier containerized application consisting of a PHP web server and a MariaDB database. We created a Docker Compose file on **App Server 2** and deployed both containers with the required port mappings, volume mounts, and database configuration.

## Requirements

* Create Docker Compose file: `/opt/itadmin/docker-compose.yml`
* Deploy two services:

  * **Web Service**

    * Container name: `php_web`
    * Image: `php` (Apache tag)
    * Host port `6400` → Container port `80`
    * Volume: `/var/www/html:/var/www/html`
  * **Database Service**

    * Container name: `mysql_web`
    * Image: `mariadb:latest`
    * Host port `3306` → Container port `3306`
    * Volume: `/var/lib/mysql:/var/lib/mysql`
    * Create database `database_web`
    * Create a custom (non-root) database user with a strong password

---

# Steps We Performed


## 1. Create Docker Compose File

```bash
sudo mkdir -p /opt/itadmin

sudo vi /opt/itadmin/docker-compose.yml
```

---

## 2. Add Docker Compose Configuration

```yaml
version: "3.8"

services:
  web:
    image: php:apache
    container_name: php_web
    ports:
      - "6400:80"
    volumes:
      - /var/www/html:/var/www/html

  db:
    image: mariadb:latest
    container_name: mysql_web
    ports:
      - "3306:3306"
    volumes:
      - /var/lib/mysql:/var/lib/mysql
    environment:
      MYSQL_DATABASE: database_web
      MYSQL_USER: webuser
      MYSQL_PASSWORD: Str0ngP@ss123
      MYSQL_ROOT_PASSWORD: RootP@ss123
```

Save and exit.

---

## 3. Deploy the Stack

```bash
cd /opt/itadmin

sudo docker compose up -d
```

---

## 4. Verify Running Containers

```bash
sudo docker ps
```

Expected output:

```text
CONTAINER ID   IMAGE          PORTS                     NAMES
xxxxxxxxxxxx   php:apache     0.0.0.0:6400->80/tcp      php_web
xxxxxxxxxxxx   mariadb:latest 0.0.0.0:3306->3306/tcp    mysql_web
```

---

## 5. Test the Web Application

```bash
curl http://stapp02:6400/
```

or

```bash
curl http://<server-ip>:6400/
```

A successful HTTP response confirms the PHP container is accessible.

# Good to Know?

### Multi-Container Applications

* Separate application and database into different containers.
* Simplify maintenance and upgrades.
* Scale services independently.
* Improve fault isolation.

### Two-Tier Architecture

* Web tier handles HTTP requests.
* Database tier stores application data.
* Services communicate over Docker's internal network.
* Keeps application components organized.

### Docker Compose

* Defines multiple services in a single YAML file.
* Starts the complete stack with one command.
* Automatically creates a network for service communication.
* Simplifies container management.

### Volume Mapping

* Persists application and database data.
* Host files are immediately available inside containers.
* Data survives container recreation.

### Environment Variables

* `MYSQL_DATABASE` creates the initial database.
* `MYSQL_USER` creates a custom database user.
* `MYSQL_PASSWORD` sets the custom user's password.
* `MYSQL_ROOT_PASSWORD` configures the MariaDB root account.
