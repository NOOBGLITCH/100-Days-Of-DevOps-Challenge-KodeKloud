# Creating a Docker Compose File

The Nautilus application development team requested a containerized Apache HTTP Server environment to host a static website. We configured Docker Compose on **App Server 3** to deploy the `httpd` container with the required port and volume mappings.

## Requirements

* Create a Docker Compose file at **`/opt/docker/docker-compose.yml`**.
* Deploy a container named **`httpd`** using the latest **httpd** image.
* Map host port **3003** to container port **80**.
* Mount host directory **`/opt/itadmin`** to container directory **`/usr/local/apache2/htdocs`**.
* Keep the container in a running state.

---

# Steps We Performed

## 1. Login to App Server 3

```bash
ssh banner@stapp03
```

---

## 2. Verify Docker Service

Before creating the container, we confirmed that Docker was running.

```bash
docker ps
```

---

## 3. Create Docker Compose Directory

```bash
sudo mkdir -p /opt/docker
```

---

## 4. Create Docker Compose File

```bash
sudo vi /opt/docker/docker-compose.yml
```

We added the following configuration:

```yaml
version: "3.8"

services:
  web:
    image: httpd:latest
    container_name: httpd
    ports:
      - "3003:80"
    volumes:
      - /opt/itadmin:/usr/local/apache2/htdocs
```

---

## 5. Deploy the Container

We started the container using Docker Compose.

```bash
cd /opt/docker

docker compose up -d
```

---

## 6. Verify Deployment

We confirmed that the container was running.

```bash
docker ps
```

Expected output:

```text
CONTAINER ID   IMAGE          PORTS                  NAMES
xxxxxxxxxxxx   httpd:latest   0.0.0.0:3003->80/tcp   httpd
```

---

## 7. Verify Port Mapping

```bash
docker port httpd
```

Expected:

```text
80/tcp -> 0.0.0.0:3003
```

---

## 8. Verify Volume Mapping

```bash
docker inspect httpd
```

The container should show:

* Host Path: `/opt/itadmin`
* Container Path: `/usr/local/apache2/htdocs`

---

# Good to Know?

### Docker Compose

* Defines multi-container applications using a YAML file.
* Simplifies container deployment and management.
* Starts all configured services with a single command.

### Service Configuration

* `image` specifies the Docker image.
* `container_name` assigns a fixed container name.
* `ports` maps host ports to container ports.
* `volumes` share files between the host and the container.

### Port Mapping

* Format: `host_port:container_port`
* Example: `3003:80` maps host port **3003** to container port **80**.

### Volume Mapping

* Shares data between the host and the container.
* Changes made on the host are immediately available inside the container.

### Common Docker Compose Commands

* Start services: `docker compose up -d`
* Stop services: `docker compose down`
* View running containers: `docker compose ps`
* View logs: `docker compose logs`
