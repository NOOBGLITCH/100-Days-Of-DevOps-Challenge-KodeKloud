# Git Branch Merge

The Nautilus application development team has been working on a project repository `/opt/demo.git`. This repo is cloned at `/usr/src/kodekloudrepos/demo` on the storage server in Stratos DC. They recently shared the following requirements with the DevOps team:

> Create a new branch `nautilus` in `/usr/src/kodekloudrepos/demo` repo from `master` and copy the `/tmp/index.html` file (present on the storage server itself) into the repo. Further, add/commit this file in the new branch and merge that branch back into the `master` branch. Finally, push the changes to the origin for both branches.

## Steps

### 1. Login into the storage server

```sh
ssh user@storage-server
```

### 2. Move into the repository

```sh
sudo su
cd /usr/src/kodekloudrepos/demo
```

### 3. Create a new branch

```sh
# ensure the current branch is master
git checkout master

# create a new branch
git checkout -b nautilus
```

### 4. Copy the file

```sh
cp /tmp/index.html .
```

### 5. Commit changes

```sh
git add index.html
git commit -m "added tmp file"
```

### 6. Push the new branch

```sh
git push -u origin nautilus
```

### 7. Switch to the master branch

```sh
git checkout master
```

### 8. Merge `nautilus` branch and push changes

```sh
git merge nautilus
git push origin master
```

## Good to Know

### Git Merge Types

* **Fast-forward**: Linear history, no merge commit.
* **Three-way Merge**: Creates a merge commit with two parents.
* **Squash Merge**: Combines all commits into a single commit.
* **Rebase**: Replays commits on top of the target branch.

### Merge Workflow

1. **Switch to Target**: Check out the branch to merge into.
2. **Merge Source**: `git merge source-branch`
3. **Resolve Conflicts**: Fix any merge conflicts.
4. **Commit**: Complete the merge if required.
5. **Push**: Upload changes to the remote repository.

### Conflict Resolution

* **Automatic**: Git merges non-conflicting changes.
* **Manual**: Developer resolves conflicting changes.
* **Merge Tools**: Use `git mergetool` for visual resolution.
* **Markers**: `<<<<<<<`, `=======`, `>>>>>>>` indicate conflicts.

### Best Practices

* **Clean Branches**: Ensure branches are up to date.
* **Test Before Merge**: Verify changes work correctly.
* **Meaningful Messages**: Write descriptive commit and merge messages.
* **Delete Merged Branches**: Clean up after a successful merge.
