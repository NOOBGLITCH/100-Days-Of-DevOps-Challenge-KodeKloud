# Configure Nginx + PHP-FPM Using Unix Sock

The Nautilus application development team planned to launch a new PHP-based application on Nautilus infra in Stratos DC. Below are the requirements and implementation steps.

## Requirements

- Install `nginx` on `app server 3`, configure it to use port `8092` and document root `/var/www/html`.
- Install `php-fpm` version `8.2` on `app server 3`, using the unix socket `/var/run/php-fpm/default.sock`.
- Configure `php-fpm` and `nginx` to work together.
- Test: `curl http://stapp03:8092/index.php`

## Steps Performed

### 1. Install nginx and php-fpm 8.2

```sh
ssh banner@stapp03
sudo dnf install -y nginx
sudo dnf module enable -y php:8.2
sudo dnf install -y php-fpm php-cli
```

### 2. Configure php-fpm pool

Created `/etc/php-fpm.d/default.conf`:

```ini
[default]
user = apache
group = apache
listen = /var/run/php-fpm/default.sock
listen.acl_users = apache,nginx
listen.allowed_clients = 127.0.0.1
pm = dynamic
pm.max_children = 50
pm.start_servers = 5
pm.min_spare_servers = 5
pm.max_spare_servers = 35
```

Disabled the default `www` pool by renaming it:

```sh
sudo mv /etc/php-fpm.d/www.conf /etc/php-fpm.d/www.conf.bak
```

### 3. Configure nginx

Created `/etc/nginx/conf.d/default.conf`:

```nginx
server {
    listen       8092;
    listen       [::]:8092;
    server_name  _;
    root         /var/www/html;
    index        index.php index.html index.htm;

    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    location ~ \.php$ {
        fastcgi_pass   unix:/var/run/php-fpm/default.sock;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        include        fastcgi_params;
    }
}
```

Cleaned up the main `/etc/nginx/nginx.conf` by removing the default server block listening on port 80, keeping only the include for `conf.d/*.conf`.

### 4. Enable and start services

```sh
sudo systemctl enable --now nginx
sudo systemctl enable --now php-fpm
```

### 5. Verification

```sh
curl http://stapp03:8092/index.php
# Output: Welcome to xFusionCorp Industries!

curl http://stapp03:8092/info.php
# Output: PHP 8.2.28 phpinfo page
```

## Good to Know

### PHP-FPM (FastCGI Process Manager)

- **Purpose**: High-performance PHP FastCGI implementation
- **Process Management**: Manages PHP worker processes
- **Performance**: Better than mod_php for high-traffic sites
- **Isolation**: Separate processes for different applications

### Communication Methods

- **Unix Sockets**: Faster, local communication only
- **TCP Sockets**: Network communication, can be remote
- **Performance**: Unix sockets generally faster for local communication
- **Security**: Unix sockets more secure (filesystem permissions)

### NGINX + PHP-FPM Benefits

- **Scalability**: Handle more concurrent connections
- **Resource Efficiency**: Lower memory usage
- **Stability**: PHP crashes don’t affect web server
- **Flexibility**: Different PHP versions per application

### Configuration Best Practices

- **Process Management**: Configure pm.max_children appropriately
- **Socket Permissions**: Ensure NGINX can access PHP-FPM socket
- **Security**: Run PHP-FPM as dedicated user
- **Monitoring**: Enable status page for monitoring
