# Deploy Redis Deployment on Kubernetes

The Nautilus application development team wanted to deploy Redis on the Kubernetes cluster as an in-memory caching solution for testing. To achieve this, we created a ConfigMap for Redis configuration and deployed a Redis Deployment with the required volumes and resource requests.

## Requirements

* Create a **ConfigMap** named `my-redis-config`
* Store `maxmemory 2mb` in the key `redis-config`
* Create a **Deployment** named `redis-deployment`
* Use image `redis:alpine`
* Container name: `redis-container`
* Replicas: `1`
* CPU request: `1`
* Mount an **EmptyDir** volume named `data` at `/redis-master-data`
* Mount a **ConfigMap** volume named `redis-config` at `/redis-master`
* Expose container port `6379`

---

# Steps We Performed

## 1. Create the Deployment and ConfigMap

Create the file **`k3s-deployment.yaml`**

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-redis-config
data:
  redis-config: |
  maxmemory 2mb
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: redis-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: redis-container
        image: redis:alpine
        ports:
        - containerPort: 6379
        resources:
          requests:
            cpu: "1"
        volumeMounts:
        - name: data
          mountPath: /redis-master-data
        - name: redis-config
          mountPath: /redis-master
      volumes:
      - name: data
        emptyDir: {}
      - name: redis-config
        configMap:
          name: my-redis-config
```

---

## 2. Apply the Manifest

```sh
kubectl apply -f k3s-deployment.yaml
```

---

## 3. Verify the Deployment

```sh
kubectl get deployments
```

Expected output:

```text
NAME               READY   UP-TO-DATE   AVAILABLE
redis-deployment   1/1     1            1
```

---

## 4. Verify the Pod

```sh
kubectl get pods
```

Expected output:

```text
NAME                                READY   STATUS
redis-deployment-xxxxxxxxxx-xxxxx   1/1     Running
```

---

## 5. Verify the ConfigMap

```sh
kubectl get configmap

kubectl describe configmap my-redis-config
```

Expected output:

```text
Name: my-redis-config

Data
====
redis-config:
----
maxmemory 2mb
```

---

## 6. Verify the Deployment Configuration

```sh
kubectl describe deployment redis-deployment
```

Verify the following:

* Image: `redis:alpine`
* Container: `redis-container`
* Replicas: `1`
* CPU Request: `1`
* EmptyDir mounted at `/redis-master-data`
* ConfigMap mounted at `/redis-master`
* Port `6379` exposed

# Good to Know?

### Redis on Kubernetes

* **In-Memory Database**: Stores data in memory for high-speed access.
* **Caching**: Improves application performance by reducing database load.
* **Data Structures**: Supports strings, hashes, lists, sets, and more.
* **Persistence**: Can optionally persist data to disk.

### ConfigMap

* Stores application configuration separately from the container image.
* Makes configuration changes easier without rebuilding images.
* Can be mounted as files or injected as environment variables.

### EmptyDir Volume

* Created when a Pod starts.
* Shared by all containers within the same Pod.
* Deleted automatically when the Pod is removed.

### Resource Requests

* Reserve the minimum CPU or memory required by a container.
* Help Kubernetes schedule Pods efficiently.

### Deployment

* Maintains the desired number of Pod replicas.
* Automatically recreates Pods if they fail.
* Supports rolling updates with minimal downtime.
