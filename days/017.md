# Day 17: Install and Configure PostgreSQL

## Objective

Create PostgreSQL users, databases, and grant privileges.

## Prerequisites

- SSH access to `stdb01` as user `peter`
- PostgreSQL installed and running
- Sudo privileges for PostgreSQL commands

## Steps Performed

### 1. Connect to the Database Server
```bash
ssh peter@stdb01
```

### 2. Switch to PostgreSQL Superuser
```bash
sudo -i -u postgres
```

This command switches to the `postgres` system user, which has superuser access to PostgreSQL.

### 3. Create Database User
```bash
psql -c "CREATE USER kodekloud_roy WITH PASSWORD 'BruCStnMT5';"
```

**What this does:**
- Creates a new PostgreSQL role/user named `kodekloud_roy`
- Sets the password to `BruCStnMT5`
- The user can now authenticate with this password

### 4. Create Database
```bash
psql -c "CREATE DATABASE kodekloud_db3 OWNER kodekloud_roy;"
```

**What this does:**
- Creates a new database named `kodekloud_db3`
- Assigns `kodekloud_roy` as the database owner
- Owner has full control over the database by default

### 5. Grant All Privileges
```bash
psql -c "GRANT ALL PRIVILEGES ON DATABASE kodekloud_db3 TO kodekloud_roy;"
```

**What this does:**
- Grants all available privileges on the database to `kodekloud_roy`
- Includes: CONNECT, CREATE, TEMPORARY, TEMP (and other schema-related privileges)

### 6. Verify Configuration

#### List All Users/Roles
```bash

```
**Expected output:**

## Verification Commands

### Test User Connection
```bash
# Exit from postgres user
exit

# Test connecting with the new user
psql -h localhost -U kodekloud_roy -d kodekloud_db3 -W
# Enter password: BruCStnMT5
```

### Check Database Ownership
```bash
sudo -i -u postgres
psql -c "\l kodekloud_db3"
```

### Verify Specific Privileges
```bash
psql -d kodekloud_db3 -c "\dp"
```
## Good to Know?

### PostgreSQL Fundamentals

- **ACID Compliance**: Atomicity, Consistency, Isolation, Durability
- **Multi-Version Concurrency Control (MVCC)**: Handle concurrent transactions
- **Extensibility**: Custom data types, functions, operators
- **Standards Compliance**: SQL standard compliant

### User and Role Management

- **Roles**: PostgreSQL uses roles instead of separate users/groups
- **LOGIN**: Role attribute allowing database connections
- **Superuser**: Full administrative privileges
- **Inheritance**: Roles can inherit privileges from other roles

### Database Permissions

- **GRANT ALL**: Full privileges on database
- **Specific Grants**: SELECT, INSERT, UPDATE, DELETE, CREATE
- **Schema Permissions**: Control access to schemas within database
- **Table-level**: Fine-grained permissions on individual tables

### Best Practices

- **Principle of Least Privilege**: Grant minimum required permissions
- **Role-based Access**: Use roles for permission management
- **Connection Limits**: Set connection limits for users
- **Password Policies**: Enforce strong password requirements
